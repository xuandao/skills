# 每日深度总结技能 (Daily Summary Agentic Workflow)

该技能采用 **Agentic Workflow** 模式。脚本仅负责最基础的文件维护，所有的分析、总结、日程安排及研究建议均由 Agent 直接读取知识库内容后自主生成。

## 运行方式

通过 openClaw 的 Cron 插件定时触发，或直接对 Agent 下令。

### 触发指令
- "执行每日深度总结任务"
- "分析心率数据"
- "检查心率健康"

## Agent 执行指南 (指令规则)

当收到执行指令时，Agent **必须** 严格按照以下步骤自主完成任务：

### 1. 基础维护 (基础动作)
- 运行 `scripts/daily_manager.py`。
- 获取输出中的 `today_note_path` (今日笔记路径)、`obsidian_root` (库根目录)、`date` (今日日期与星期) 以及 `weather` (今日天气)。

### 2. 知识获取 (自主阅读)
- **获取动态**: 使用 `run_shell_command` 在 `obsidian_root` 中运行 `find . -name "*.md" -mtime -3`，找出近 3 天修改的文件。
- **获取运动数据**: 使用 `run_shell_command` 在 `obsidian_root` 中运行 `find Areas/Running -name "*.md" -mtime -7`，找出近 7 天的运动笔记并阅读。
- **深度阅读**: 使用 `read_file` 阅读上述文件的内容（优先阅读最近修改的 5-10 篇）。
- **获取收藏**: 阅读 `Resources/收藏夹` 目录下最新的 1-2 篇笔记。
- **系统状态**: 运行 `openclaw sessions --all-agents --active 1440 --json` 获取过去 24 小时的 Token 消耗。

### 3. 心率健康分析
当用户要求分析心率数据或执行每日总结时，运行心率分析脚本：

```bash
python3 scripts/heart_rate_analyzer.py
```

该脚本会：
- 从 Garmin 跑步笔记中提取**运动心率**数据
- 从 Garmin 日常健康数据中提取**静息心率、HRV、血氧、呼吸频率**等
- 自动检测以下健康信号：

#### 🔴 紧急信号（收到就停下来，立刻就医）
1. **运动中突然心率飙到 200+** → 立即停止运动，坐下休息，不缓解就叫救护车
2. **运动中骤降到 50 以下** → 立即停止运动，就医检查

#### 🟡 早期预警（天/周级别变化，尽快约检查）
3. **HRV 连续 3 天暴跌 >30%**
   - 正常波动：±15%
   - 危险信号：从 80ms 连续降到 50ms 以下
4. **静息心率持续 >100 或 <40**
   - 连续3天 >100 bpm → 心动过速，需要查
   - 连续3天 <40 bpm → 心动过缓，需要查
5. **血氧均值跌破 90%** → <90% = 严重缺氧，夜间心律失常风险大增

#### 🟢 长期趋势监测（月级别，定期关注）
6. **HRV 长期趋势持续下行** → 如果 3 个月趋势从 90→70→50，即使没有急性事件 → 说明自主神经功能在退化，做心脏检查
7. **呼吸频率持续升高**
   - 正常：12-16 次/分
   - 如果持续升到 18-20+，可能提示心功能下降

**心率分析结果处理**：
- 如果有 🔴 紧急信号：立即通过 Telegram 发送警报
- 如果有 🟡 早期预警：在 Daily 笔记中添加心率健康章节，并在 Telegram 简报中提及
- 如果有 🟢 长期趋势：仅在 Daily 笔记中记录

### 4. 智能分析与生成 (逻辑核心)
Agent 需根据读到的内容，自主生成以下两个部分：

#### [部分 A：微信极简简报]
- **要求**: 极其简洁，总长度 < 200 字。
- **包含**: 
    - 📅 **日期**: 使用输出中的 `date`。
    - ⛅️ **天气**: 使用输出中的 `weather`。
    - 🌟 **晨间灵感**: 一句诗或格言。
    - 📅 **核心日程**: 1-3点建议。
    - ❤️ **心率健康**: 如有预警或紧急信号，简要提示。
    - 🤖 **系统消耗**: 过去 24h Token 消耗。
    - 📚 **1条核心收藏**: 推荐一篇收藏夹内容。
- **结尾**: 提示"详细深度洞察已同步至 Daily 笔记"。
- **发送**: 通过 Telegram 推送到用户（使用 `channel: telegram`，`target` 为 `5247154884`）。

#### [部分 B：Daily 深度洞察]
- **要求**: 专业、深刻。
- **内容**:
    - **近 3 日动态回顾**: 梳理最近关注的重点 (Focus)、新想法 (Ideas) 和心得 (Insights)。
    - **🏃 运动表现与安排**:
        - **回顾**: 总结近 7 天的运动频率、强度（配速、心率）及身体反馈。
        - **计划**: 结合当前体能状态与天气/日程，给出今日或本周后续的运动类型建议（如：节奏跑、轻松跑或休息）。
    - **❤️ 心率健康分析**: 整合心率数据分析脚本的输出，包括警报统计、详细分析和建议。
    - **任务分析**: 结合昨日遗留任务与笔记内容，提供今日重点推进事项的深度分析。
    - **调研建议**: 提供 2-3 条开放性的思维建议或前瞻性方案调研思路。
- **写入**: 将此部分内容以 `## 🧠 AI 深度洞察与调研建议` 为标题，**追加 (Append)** 到 `today_note_path` 文件的末尾。

## 心率数据存储规范

### Garmin 日常健康数据
- 位置：`Areas/Health/Garmin/YYYY-MM-DD.md`
- 数据来源：Garmin Connect API（静息心率、HRV、睡眠、身体电量等）
- 获取脚本：`scripts/fetch_garmin_daily_health.py`
- 数据格式：
  ```markdown
  # Garmin 健康数据 - 2026-03-25

  ## 心率数据
  - 静息心率: 52 bpm
  - 睡眠平均心率: 48 bpm
  - 全天最高心率: 145 bpm
  - 全天最低心率: 42 bpm

  ## HRV
  - 夜间平均: 72 ms
  - 周平均: 68 ms
  - 状态: BALANCED

  ## 睡眠
  - 时长: 7.5 小时
  - 血氧平均: 98.5%
  - 呼吸频率: 14.2 次/分

  ## 身体电量
  - 晨起: 65
  - 最高: 95
  - 最低: 30
  ```

### Garmin 跑步数据
- 位置：`Areas/Running/<训练类型>/YYYY-MM-DD-<训练类型>-<序号>.md`
- 心率数据格式：
  ```markdown
  - 平均心率: 145 bpm
  - 最大心率: 168 bpm
  ```
- 由 `garmin-running` skill 自动生成

## 配置说明 (config.json)

- **OBSIDIAN_ROOT**: Obsidian 库的绝对路径。
- **DAILY_FOLDER**: 每日笔记相对路径。
- **ARCHIVE_FOLDER**: 归档相对路径。
- **TEMPLATE_PATH**: 每日笔记模板路径。

## 脚本说明

### daily_manager.py
- 归档 7 天前的笔记
- 提取未完成任务
- 创建/更新今日笔记
- 获取天气和日历事件

### heart_rate_analyzer.py
- 解析 Garmin 运动心率数据
- 解析 Garmin 日常健康数据（静息心率、HRV、血氧、呼吸频率）
- 检测紧急信号、早期预警、长期趋势
- 生成 Markdown 格式的心率健康报告

### fetch_garmin_daily_health.py
- 从 Garmin Connect API 获取日常健康数据
- 支持数据类型：
  - 静息心率（`restingHeartRate`）
  - 全天候心率趋势（`dailyHeartRate`）
  - HRV 数据（夜间平均、周平均、基线）
  - 睡眠数据（含睡眠心率、血氧、呼吸频率）
  - Body Battery（身体电量）
  - 压力数据
  - 步数数据
- 使用方式：
  ```bash
  python3 fetch_garmin_daily_health.py today
  python3 fetch_garmin_daily_health.py 2026-03-25
  python3 fetch_garmin_daily_health.py yesterday /path/to/output.json
  ```
