# 一卡通账户变动通知邮件解析规则

## 邮件格式分析

基于对最近 30 天（约 200+ 封）邮件的分析，招商银行一卡通账户变动通知邮件主要有以下两种格式：

---

## 格式一：快捷支付

### 邮件内容格式

#### 完整格式（带余额）
```
您账户 XXXX 于 MM 月 DD 日 HH:mm 在 [商户名称] 快捷支付 XX.XX 元，余额 XX.XX
```

#### 无余额格式
```
您账户 XXXX 于 MM 月 DD 日 HH:mm 在 [商户名称] 快捷支付 XX.XX 元
```
（某些邮件可能没有"余额"字段，可能是银行限制或邮件截断）

### 示例

```
完整格式：
您账户 9583 于 03 月 13 日 09:44 在支付宝 - 蚂蚁（杭州）基金销售有限公司快捷支付 10.00 元，余额 4506.81

无余额格式：
您账户 9583 于 03 月 05 日 10:23 在支付宝 - 淘宝（中国）软件有限公司快捷支付 199.00 元
```

### 正则表达式

```python
# 完整格式（带余额）
pattern = r'账户.*?(\d{4}).*?(\d{2})\s*月\s*(\d{2})\s*日\s*(\d{2}):(\d{2}).*?在\s*(.+?)\s*快捷支付\s*([\d.]+)\s*元.*?余额\s*([\d.]+)'

# 无余额格式
pattern = r'账户.*?(\d{4}).*?(\d{2})\s*月\s*(\d{2})\s*日\s*(\d{2}):(\d{2}).*?在\s*(.+?)\s*快捷支付\s*([\d.]+)\s*元'
```

### 提取字段

| 组号 | 字段 | 示例值 |
|------|------|--------|
| 1 | 账户尾号 | 9583 |
| 2 | 月份 | 03 |
| 3 | 日期 | 13 |
| 4 | 小时 | 09 |
| 5 | 分钟 | 44 |
| 6 | 商户名称 | 支付宝 - 蚂蚁（杭州）基金销售有限公司 |
| 7 | 交易金额 | 10.00 |
| 8 | 账户余额 | 4506.81（完整格式有，无余额格式缺失） |

### 注意事项

1. **空格处理**: 邮件内容中各字段之间有多个空格，正则需使用 `\s*` 宽松匹配
2. **括号**: 商户名称中可能包含全角括号 `（）` 或半角括号 `()`
3. **商户名称**: 可能包含空格，使用非贪婪匹配 `.+?`

## 格式二：定投扣款

### 2.1 标准格式

#### 邮件内容格式

```
您尾号 XXXX 的账户于 MM 月 DD 日执行「[产品名称]」的定投计划，扣款 XX 元，活期余额 XX.XX 元
```

#### 示例

```
您尾号 9583 的账户于 03 月 13 日执行「大成标普 500 等权重指数 (QDII)A 人民币」的定投计划，扣款 50 元，活期余额 4696.81 元
```

#### 正则表达式

```python
pattern = r'尾号\s*(\d{4}).*?的账户于\s*(\d{2})\s*月\s*(\d{2})\s*日\s*执行「(.+?)」的定投计划.*?扣款\s*([\d.]+)\s*元.*?活期余额\s*([\d.]+)\s*元'
```

### 2.2 简化格式

#### 邮件内容格式

```
您尾号为 XXXX 的账户于 MM 月 DD 日定投「[产品名称]」XX 元，活期余额 XX.XX 元
```

#### 示例

```
您尾号为 0707 的账户于 03 月 11 日定投「摩根标普 500 指数 (QDII) 人民币 A」50 元，活期余额 8534.95 元
```

#### 正则表达式

```python
# 支持"尾号为 XXXX"格式
pattern = r'尾号为\s*(\d{4})\s*的账户于\s*(\d{2})\s*月\s*(\d{2})\s*日定投「(.+?)」\s*([\d.]+)\s*元.*?活期余额\s*([\d.]+)\s*元'

# 支持"尾号 XXXX"格式（没有"为"字）
pattern = r'尾号\s*(\d{4})\s*的账户于\s*(\d{2})\s*月\s*(\d{2})\s*日定投「(.+?)」\s*([\d.]+)\s*元.*?活期余额\s*([\d.]+)\s*元'
```

### 提取字段

| 组号 | 字段 | 示例值 |
|------|------|--------|
| 1 | 账户尾号 | 9583 |
| 2 | 月份 | 03 |
| 3 | 日期 | 13 |
| 4 | 产品名称 | 大成标普 500 等权重指数 (QDII)A 人民币 |
| 5 | 扣款金额 | 50 |
| 6 | 活期余额 | 4696.81 |

### 注意事项

1. **产品名称**: 可能包含特殊字符如 `()`、中文括号等
2. **余额字段**: 使用"活期余额"区别于快捷支付的"余额"
3. **格式变体**: 存在"尾号为 XXXX"和"尾号 XXXX"两种变体

## 邮件查询

### Gmail 搜索语法

```
一卡通账户变动通知 newer_than:Xd
```

其中 `X` 为天数，如：
- `newer_than:1d` - 最近 1 天
- `newer_than:7d` - 最近 7 天（周）
- `newer_than:30d` - 最近 30 天（月）

### gws 命令示例

```bash
# 查询最近 7 天的邮件
gws gmail users messages list --params '{
  "userId": "me",
  "q": "一卡通账户变动通知 newer_than:7d",
  "maxResults": 100
}'

# 获取邮件详情
gws gmail users messages get --params '{
  "userId": "me",
  "id": "邮件 ID",
  "format": "full"
}'
```

## 解析流程

1. **查询邮件列表**: 使用 gws 查询符合条件的邮件
2. **获取邮件内容**: 逐封获取邮件完整内容
3. **Base64 解码**: Gmail API 返回的是 URL-safe Base64 编码
4. **正则匹配**: 尝试匹配两种格式
5. **数据提取**: 提取交易字段
6. **数据排序**: 按日期和时间排序

## 数据处理规则

## 数据处理规则

### 金额处理

- 快捷支付：金额为正数，表示支出
- 定投扣款：金额为正数，表示扣款
- 两种方式都从账户扣除，计算总支出时相加

### 余额处理

- 使用最后一笔交易的余额作为期末余额
- 可通过余额变化验证交易完整性

### 账户过滤

配置 `account_filter` 可选择性处理特定账户：

```python
CONFIG = {
    "account_filter": ["9583", "0707"]  # 只处理这两个账户
}
```

### 文件输出规则

- **月度账单文件**: 统一使用 `YYYYMM-一卡通账单.md` 格式
- **追加模式**: 每次运行脚本时，新内容会追加到当月文件末尾
- **分隔符**: 多次运行之间使用 `---` 分隔

---

## 已知限制

1. **邮件依赖**: 依赖银行发送邮件，如有遗漏无法补全
2. **格式变更**: 如银行更改邮件格式，需更新正则表达式
3. **时间精度**: 定投扣款邮件通常不包含具体时间
4. **重复邮件**: 某些交易可能触发多封邮件（如定投确认）
